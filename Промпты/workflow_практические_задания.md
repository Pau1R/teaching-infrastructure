# Промпт для генерации практических заданий урока

Сгенерируй индивидуальные практические задания для студента в формате Markdown. Учитывай данные урока и текущий уровень студента. Пиши кратко и технически. Без лишнего текста.

## Входные данные
- Номер урока: `<lesson_number>`
- Тема урока: `<lesson_topic>`
- Номер задания: `<task_number>`

## Общие правила
- Задание должно соответствовать теме урока и развивать связанные навыки.
- Сложность должна быть немного выше текущего уровня студента. Но задание должно быть выполнимым.
- Если студент слабый по данным `performance.md` и `all_performance.md`. Дай больше структурированных подсказок и промежуточных ориентиров.
- Если студент сильный. Уменьшай количество подсказок. Добавляй небольшие расширения по желанию.
- Форматируй результат как один законченный Markdown файл задания.

## Как учитывать уровень студента
1. Проанализируй `performance.md`.
2. Учитывай сводку из `all_performance.md`.
3. Определи уровень: слабый, средний, сильный.
4. Подбери сложность и объем подсказок.

## Формат результата
Сформируй один Markdown документ со следующей структурой. Заполни все секции.

### Заголовок
`Задание <task_number>. <lesson_topic>`

### Описание
Короткое. конкретное описание задачи в 2–4 предложения.

### Пример входных данных
Минимально достаточный пример. Можно несколько кейсов, если нужно.

### Пример выходных данных
Ожидаемый результат для приведенных входных данных.

### Критерии успешного выполнения
- Список четких критериев проверки. 3–6 пунктов.

### Подсказки
- Если уровень слабый. Дай 3–6 подсказок по шагам с нарастающей конкретикой.
- Если уровень средний. Дай 2–4 подсказки среднего уровня абстракции.
- Если уровень сильный. Дай 1–2 общих направления без явных решений.

### Частые ошибки и ограничения
- Укажи 2–5 типичных ошибок для этой темы.
- Явно укажи ограничения. например. форматы ввода. граничные случаи. запреты на использование нецелевых библиотек.

## Требования к стилю
- Пиши короткими фразами. техническим языком.
- Без воды. только суть.
- Примеры кода. если нужны. оформляй блоками ```lang.

## Примечания
- Задание номер и тема должны строго соответствовать входным данным.
- Все разделы обязательны.
- Если тема предполагает несколько подходов. упомяни допустимые альтернативы в критериях или подсказках.
