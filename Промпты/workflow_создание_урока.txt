---
description: Автоматическое создание структуры урока и материалов на основе course.md в python2026_tasks с последующей генерацией файлов и проверками
---

Тебе нужно автоматически создать структуру урока по данным из файла
course.md в репозитории python2026. Выполняй ровно по шагам. Не
объединяй шаги. Не пропускай шаги. Если какой‑то файл или папка уже
существует, пропускай их создание и переходи дальше.

Используй урок с датой сегодня. Если его нет, выбери следующую доступную
дату. Прошедшие уроки игнорируй.

Параметры:
- Репозиторий задач: python2026_tasks.
- Репозиторий преподавателя: python2026.
- Источник данных: файл course.md.
- Выбор даты: локальная дата системы. Если подходящих уроков нет. остановись и сообщи.

Общие правила:
- Выполняй шаги строго по порядку. Не объединяй. Не пропускай.
- Ничего не удаляй. Не перезаписывай существующие файлы.
- Если имя файла или папки содержит запрещенные символы. замени их на пробел.
- Все пути и имена проверяй перед созданием. при ошибке остановись и сообщи.

------------------------------------------------------------------------

ШАГ 1. Создание структуры урока в репозитории python2026_tasks

1.  Прочитай course.md и определи:

    -   номер урока
    -   название темы
    -   дату урока
    -   количество практических заданий

2.  В папке lessons создай папку урока. Формат имени:
    lesson <номер урока> (<дата урока>). <Название урока на русском> Если такая
    папка уже существует, пропусти создание.

3.  Внутри папки урока создай:

    -   подпапку tasks

4.  В tasks создай подпапки 1, 2, 3, … Количество должно совпадать с
    количеством практических заданий. Пропускай те, которые уже есть.

5.  Создай файл конспекта:
    notes.md Если файл существует, не перезаписывай, а пропусти.

6.  Создай файл теста: test <номер урока>.json Если уже существует,
    пропусти. Требования:

    -   10 вопросов по теме урока
    -   6 вариантов ответа в каждом
    -   один правильный вариант = 1 балл
    -   все варианты ответов кроме шестого должны быть перемешаны
    -   остальные варианты = 0 баллов
    -   шестой вариант должен быть: «Не знаю ответа», он дает 0.2 балла
    Примечание. Если формат теста не задан в проекте. используй JSON:
    ```json
    [
      {
        "question": "Текст вопроса",
        "options": ["Вариант A", "Вариант B", "Вариант C", "Вариант D", "Вариант E", "Не знаю ответа"],
        "correct_index": 0,
        "scores": [1, 0, 0, 0, 0, 0.2]
      }
    ]
    ```

------------------------------------------------------------------------

ШАГ 2. Проверка и исправления

1.  Проверь файл урока на полноту и соответствие теме. Исправь при
    необходимости.
2.  Проверь тест на технические ошибки и соответствие теме. Исправь при
    необходимости, но не удаляй и не пересоздавай файл, если он уже был.

------------------------------------------------------------------------

ШАГ 3. Создание подробного содержания урока в репозитории python2026

1.  В репозитории преподавателя python2026 создай папку урока с тем же
    названием, что и в python2026_tasks. Если уже есть, пропусти.
2.  Внутри создай файл с тем же именем, что в python2026_tasks. Если
    существует, пропусти.
3.  Заполни файл подробным содержанием темы по правилам из
    /lesson-content.

------------------------------------------------------------------------

ШАГ 4. Генерация индивидуальных материалов студентам

Для каждого студента в папке students:

1.  Прочитай performance.md внутри папки студента.

2.  Прочитай all_performance.md внутри общей папки students.

3.  Учитывая оба файла, сделай следующее:

    Задания урока: В каждой подпапке tasks/<n> создай файл:
    task <номер урока>.<номер задания> <имя студента>.md Пропускай
    существующие файлы. Содержание по правилам /student-tasks.

------------------------------------------------------------------------

ШАГ 5. Валидация результата

1. Проверь соответствие структуры и количества заданий данным из course.md.
2. Убедись, что существующие файлы не перезаписывались. при наличии пропускали создание.
3. Проверь корректность имен файлов и папок. недопустимые символы заменены на пробел.
4. Проверь, что тест содержит 10 вопросов и в каждом 6 вариантов. шестой вариант «Не знаю ответа».

------------------------------------------------------------------------

ШАГ 6. Коммит и отправка изменений

1.  Подготовь коммит со всеми созданными и измененными файлами.
2.  Сообщение коммита:
    lesson <номер урока>: generated lesson structure, tests and tasks for all students
3.  Отправь коммит в соответствующие удаленные репозитории.
